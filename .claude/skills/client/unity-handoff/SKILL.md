# 유니티 에디터 작업 핸드오프

> IDE 클로드(코드 작업) → 유니티 어시스턴트(에디터 작업) 인계를 매끄럽게 하기 위한 규칙.
> 코드 변경만으로 끝나지 않고 에디터 측 후속 작업이 남을 때, 답변 말미에
> **복사용 프롬프트**를 자동으로 첨부한다.

---

## 트리거 — 언제 프롬프트를 만드는가

IDE 클로드의 작업 결과로 **유니티 에디터에서 사람이/어시스턴트가 해야 하는 후속 작업**이
생겼을 때, 답변 마지막에 핸드오프 프롬프트를 붙인다. 대표 예:

- 인스펙터 값 설정·컴포넌트 연결 (직렬화 필드 추가/변경 등)
- SO(`.asset`) 데이터 생성·수정 (새 카드/스킬/아이템 데이터 등)
- 프리팹·씬 구성 변경
- 임시 그래픽 에셋 제작
- **에디터 동기화가 필요한 변경** — IDE/파일 탐색기 등 유니티 밖에서 에셋을
  옮기거나 이름을 바꾼 경우 (Unity가 참조·메타·`ProjectSettings`를 자동 갱신해야 함)

## 하지 않을 것

- 순수 코드 작업만으로 완결되어 **에디터 개입이 불필요하면 프롬프트를 만들지 않는다.**
  (불필요한 첨부 금지)
- 사용자가 직접 하는 것이 더 빠른 사소한 클릭 한두 번은 본문 안내로 충분하다.

---

## 프롬프트 작성 규칙

1. **복사 편의** — 전체를 코드블록(```)으로 감싼다.
2. **정확한 경로/값** — 대상 파일·오브젝트의 경로와 설정할 값을 구체적으로 명시한다.
   (예: `Assets/Scripts/SO/SkillSO.asset`의 `skills` 배열)
3. **전제 설명** — 코드 변경으로 생긴 전제를 함께 적는다.
   (예: "새 int 필드는 직렬화 시 기존 항목에 기본값 0이 들어가므로 직접 값을 지정해야 함")
4. **그래픽 요청** — 저장 폴더 위치와 "기존 에셋 포맷(크기·형태·스타일)에 맞추라"는
   지시를 포함한다. (`AssistantCustomInstructions.txt`와 일관)
5. **실행 가능 형태** — 한국어로, 작업이 여러 개면 `[1] [2] …` 단계로 구분한다.

---

## 예시

> (직전 "버프 지속시간" 작업의 핸드오프를 축약한 형태)

```
SkillSO.asset(Assets/Scripts/SO/SkillSO.asset)의 skills 배열을 편집해줘.
Skill에 int 필드 buffDuration이 추가되었다 (0=영구, 1=이번 턴만, N=내 턴 N회).
새 필드라 기존 항목에는 기본값 0이 들어가 있으니 직접 지정해야 한다.

[1] "전열 강화", "집중 강화"의 buffDuration = 1 로 설정
[2] 모든 스킬 description을 더 자세히 다시 작성 (지속시간 명시)
[3] 영구 버프 카드 "영구 각성"(BuffSingle, manaCost 5, value 4, buffDuration 0) 추가
    - 아이콘은 Assets/Contents/Sprites/Skills/ 에 기존 스킬 아이콘과 같은 포맷으로 제작
```

## 에디터 동기화 안내 (마지막에 항상 확인)

이번 작업으로 **유니티 밖(IDE·파일 탐색기 등)에서 에셋을 옮기거나 이름을 바꿨거나,
Unity가 자동 갱신해야 할 참조(`ProjectSettings`·`.meta`·씬·SO 등)가 생겼다면**,
답변 말미에 아래 안내를 반드시 넣는다.

> ⚠️ **유니티 에디터를 한 번 열어(포커스를 줘) 동기화시킨 뒤 커밋하세요.**
> 밖에서 옮긴 에셋은 Unity가 켜질 때 참조·메타·빌드 세팅을 자동으로 갱신합니다.
> 동기화로 바뀐 `ProjectSettings/` 등의 변경분까지 **함께 커밋**해야
> 경로 어긋남·stash 충돌을 막을 수 있습니다.

> **배경:** 과거에 씬 파일을 유니티 밖에서 `Assets/Scenes/`로 옮긴 뒤
> `EditorBuildSettings.asset`의 경로 갱신을 빼먹고 커밋해, 협업자 커밋과
> 동일한 한 줄 수정이 충돌하며 stash가 발생한 사례가 있었다.
